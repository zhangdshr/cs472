4. What is "Unobtrusive Javascript"? What is the practical application of Unobtrusive Javascript (and the reasons for using it)?
It means the separation of content (HTML) presentation(CSS) and behavior (Javascript).
In the lecture slide code, we attach event handlers to elements' DOM objects in the JavaScript code, instead of HTML is cluttered with JS code.
Usability: An unobtrusive DOM script does not draw the attention of the user - visitors use it without thinking about it. Graceful degradation: Unobtrusive DOM scripts never generate error messages, in any browser, even when they fail. If features cannot be presented properly, they silently disappear. Accessibility: If any script fails, the page still delivers its core functions and information via the markup, stylesheets and/or server-side scripting. Separation: For the benefit of other and future web developers, all JavaScript code is maintained separately, without impacting other files of script, markup or code.
